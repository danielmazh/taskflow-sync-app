# Closed Testing — 12 testers / 14 continuous days plan

Required for **new personal Google Play developer accounts** registered after Nov 13 2023: you must run a closed test with **at least 12 testers opted in continuously for 14 days** before your app can be promoted to **Production**. There is no API; every step is a manual click in Play Console.

Console path: **Play Console → All apps → TaskFlow Sync → Testing → Closed testing → Create track**.

---

## Pre-flight checklist (before recruiting testers)

- [ ] Play developer account created and identity-verified (D-U-N-S not needed for personal accounts; government ID is).
- [ ] App created in Play Console; package name matches Phase 4 build's `applicationId`.
- [ ] First AAB uploaded to the closed track. **Use Play App Signing** (Google recommendation; required for new apps since Aug 2021). Capture the **App Signing SHA-1** from Play Console → Setup → App signing — you will need it for the Android OAuth client.
- [ ] Pre-launch report shows no crashes or critical warnings.
- [ ] Store listing + data safety + content rating + target audience are all complete (drafts are fine but the track must be set up).
- [ ] Privacy policy is live at `{{PRIVACY_URL}}`.

## Track configuration

| Field | Value |
|---|---|
| Track name | `closed-12` |
| Testing method | **Closed testing** |
| Tester list type | **Google Groups** (recommended — easier to manage than email-by-email; also accepts a list of emails) |
| Feedback URL or email | `{{SUPPORT_EMAIL}}` |
| Countries / regions | Same set you plan to launch in for Production |

## Tester recruitment (target ≥12 active testers, plan for ≥15 to allow for drops)

| Channel | How |
|---|---|
| Personal network | Family / friends / colleagues with Android devices. Easiest to keep them engaged for 14 days. |
| Reddit r/androidtest | A standing thread for exactly this. Post with a clear opt-in link. |
| Tester-recruitment services | r/androidtest_2 or paid services like TestApp.io. Lower bar but lower engagement. |

### What you send each tester

```
Subject: Help me test TaskFlow Sync (closed Play test, 14 days)

Hi {{NAME}},

I'm shipping a small Android task manager called TaskFlow Sync and Google requires
12 testers for 14 continuous days before I can publish.

To join:
1. Open this link on your Android phone: {{OPT_IN_URL}}
   (You'll see "Become a tester". Tap it.)
2. Install the app from Play. It will be listed as "TaskFlow Sync (beta)".
3. Use it whenever you'd normally use a task app. That's it.
4. Please leave it installed for 14 days. If you find a bug or have a comment,
   reply to this email or use the in-app support link.

Thank you!
```

`{{OPT_IN_URL}}` is generated by Play Console after you create the track — it's something like `https://play.google.com/apps/internaltest?...`.

## During the 14 days

- [ ] Monitor **Play Console → Testing → Closed testing → Track manager** for the daily **active testers** count. The count must stay ≥12 every day. Drops below 12 reset the clock.
- [ ] Respond to tester feedback within 24h. Google looks at this when you apply to promote.
- [ ] Ship a new AAB only if you must (bug fix). Re-using the same track is fine; the 14-day window does not reset on a new build.
- [ ] On day 14, take a screenshot of the active-testers chart for your records.

---

## After 14 days — promote to Production

- [ ] **Play Console → Apply for production access** (Testing → Choose a testing track → Apply for production).
- [ ] Fill the application: "What did testing teach you?", "How did you select testers?", "Show evidence of feedback".
- [ ] Reviewer SLA: a few days to ~2 weeks.
- [ ] If approved, promote the build from closed-12 → production (Play Console → Production → Create new release → Choose from existing track).

---

## Common pitfalls

- Testers must **opt in** via the URL, not just install the app. Installs without opt-in don't count.
- Testers must have an Android device with Google Play installed. Emulators don't count toward the 12.
- The 12 testers must be **unique Google accounts**. One tester with two accounts ≠ two testers.
- If you also want OAuth verification approved by the time you launch, **start it now in parallel** — verification can take 2–6 weeks (see `docs/oauth-verification.md`).
