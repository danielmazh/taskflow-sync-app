Insight: Phase 4c hardens Phase 4b's explicit Calendar UX. Three changes. (1) Add-task sheet shows the "Add to Google Calendar" checkbox at creation (whenever `!_isEdit && calendarAvailable`); it is *disabled* with the subtitle "Pick a due time to add it to your calendar" until a time is picked. `addToCalendar` in the result is only ever true when both conditions hold. (2) `CalendarSyncService.upsertEvent` now treats a patch result with `status == 'cancelled'` as gone and re-inserts a fresh event (Google returns the cancelled shape for a while after a manual deletion before 410 takes over). New `eventLinkStatus(eventId) → EventLinkStatus.{exists,gone,unknown}` does a definitive `events.get` check; gone = 404/410/cancelled, unknown = auth/network/quota. New `TaskStore.reconcileCalendarLinks({minInterval, force})` walks every linked task, queries the service, and clears `calendarEventId` **only on `gone`** (never on `unknown` — offline must not orphan). Wired to the lifecycle resume hook in `main.dart` (replaces `rehydrate` direct call → `onResume` which awaits rehydrate then unawaited-fires reconcile). In-memory throttle = 60s; in-flight de-dupe; `force` bypass for tests. (3) AppBar replaced `_CalendarMenuButton` (calendar glyph) with `_AccountButton`: signed-out → `account_circle_outlined` → tap = `connect()`; signed-in → `CircleAvatar` of `user.photoUrl` (initials fallback) → menu with avatar+name+email, Google Calendar row (authorized/not-authorized + grant-access shortcut), and Disconnect. `CalendarConnection` gained `photoUrl: String?` populated from `user.photoUrl` in `_onAuthEvent` and `connect()`. Session restore via `attemptLightweightAuthentication()` + persisted `calendar_connected` flag is unchanged — already correct from Phase 4 design; the GCP "Testing"-mode ~7-day refresh-token TTL re-auth cadence is an unverified-app limit (memory 019), not a regression. Tests 73 → 82, all green; analyze clean.
Rule: Reconcile MUST distinguish `gone` from `unknown`. Clearing `calendarEventId` on `unknown` would silently orphan every linked task during an offline resume. The `cancelled` patch-result branch in `upsertEvent` must stay co-located with the 404/410 branch — both paths mean "event is effectively gone; insert a new one."
