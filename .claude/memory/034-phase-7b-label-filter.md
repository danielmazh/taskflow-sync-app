Insight: Phase 7b — label filtering on Home is view-state only, lives entirely in HomeScreen as ephemeral 'String? _activeLabel'. NEVER in TaskStore, NEVER persisted, resets on app restart. The 'Unlabeled' chip uses an internal sentinel constant '_kUnlabeledSentinel' (a string starting with a space — impossible as a trimmed user label) so a real label literally named 'Unlabeled' cannot collide. Filter applies to active tasks BEFORE _group(...), and the _SummaryLine (overdue/due-today counts) MUST be computed off the filtered set so the numbers match what's visible. Strip is shown via SliverToBoxAdapter (simpler than a pinned sliver) ONLY when store.labels is non-empty; it stays visible while filtering even when the filtered list is empty so the user can switch or clear. Empty-filtered state is a dedicated _LabelFilterEmpty widget with a 'Show all' TextButton — NOT the existing nothing-yet/all-done states. Match is case-insensitive on trimmed label. Pre-existing Phase 7a TaskCard chip test needed find.descendant scoping because the strip also renders the same label text.
Rule: Label filter/search remain HomeScreen view state — never TaskStore, never persisted. _kUnlabeledSentinel must be unforgeable by user input (lead with whitespace). Filtered counts MUST reflect the filtered set, not the global set.
