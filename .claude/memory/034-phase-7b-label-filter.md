Insight: Phase 7b — label filter on Home is view-state only: ephemeral `String? _activeLabel` in HomeScreen. NEVER in TaskStore, NEVER persisted, resets on app restart. The Unlabeled chip uses sentinel `_kUnlabeledSentinel` (a string with a leading space — unforgeable by trimmed user input) so a real label named "Unlabeled" can't collide. Filter applies to active tasks BEFORE `_group(...)`; `_SummaryLine` counts must be computed off the filtered set so visible numbers match. Strip is a SliverToBoxAdapter, shown only when `store.labels` is non-empty, stays visible while filtering even when filtered list is empty (so the user can switch/clear). Dedicated `_LabelFilterEmpty` widget with a "Show all" TextButton — NOT the existing nothing-yet/all-done states. Match is case-insensitive on trimmed label. The Phase 7a TaskCard chip test needed `find.descendant` scoping because the strip surfaces the same label text.
Rule: Label filter/search are HomeScreen view state — never TaskStore, never persisted. `_kUnlabeledSentinel` must be unforgeable (lead with whitespace). Filtered counts must reflect the filtered set, not the global set.
