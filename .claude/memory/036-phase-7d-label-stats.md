Insight: Phase 7d — Statistics per-label breakdown is a thin pure layer on top of the existing StatsData. New `LabelStat` value-type + top-level `labelStatsFrom(tasks, now)` in lib/util/stats_data.dart. Buckets tasks by trimmed, case-insensitively-deduped label (first-seen casing preserved), then runs `StatsData.from` on each subset for active/completed/onTimePct/hasOnTimeData — NO parallel outcome math. Order: real labels alphabetically (case-insensitive) then an `Unlabeled` bucket last, only when (a) ≥1 real label exists AND (b) ≥1 task is untagged. Returns `const []` for empty input or when every task is unlabeled — the UI then omits the whole "By label" section so a lonely Unlabeled card never appears. `statistics_screen.dart` renders a single `_ByLabelCard` (surfaceContainerHighest + AppRadius for parity with existing cards). Per-bucket numbers are unit-tested by reading `StatsData.from` on the same subset and asserting equality — guarantees we never drift from the global on-time engine.
Rule: Per-label stats MUST reuse `StatsData.from` over per-label subsets — never duplicate outcome math. The Unlabeled bucket is conditional: include only when a real label is also present AND at least one task is untagged; otherwise omit the whole section.
