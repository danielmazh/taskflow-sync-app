Insight: Phase 7c — task search is a pure widget-free helper `lib/util/task_search.dart` plus Flutter's built-in `showSearch` + a `SearchDelegate`. NO new dep. The helper returns `const []` for empty/whitespace query (search is opt-in — NOT a default "show everything"), case-insensitive substring over title + note + label, stable input order, no fuzzy/Levenshtein. `_TaskSearchDelegate` searches `widget.store.tasks` (ALL tasks, active AND completed — completed-task searchability is a hard requirement). Result rows are read-only: `TaskCard` without `onToggle`/`trailing`/dismiss; tap calls `close(context, task)` and HomeScreen reopens the edit sheet on the returned task. Search is independent of the Phase 7b filter strip — separate view tools. `buildSuggestions` and `buildResults` share the same body (no expensive computation to defer). Memory indexed 035 to dodge 034 owned by the parallel unmerged feature/phase-7b-label-filter branch.
Rule: For task search never add a dep — `showSearch` + `SearchDelegate` is the whole mechanism. Empty/whitespace query MUST return `[]`, never "all tasks". Search MUST span completed tasks too; caller chooses scope, the helper does not filter.
